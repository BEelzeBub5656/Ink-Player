## 技术决策 (已确认)

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 主控 | ESP32-PICO-V3 | PSRAM用于OTA缓冲+音频缓冲 |
| 唤醒词 | ESP-SR本地 | 无需联网, <500ms |
| 音频上传 | HTTP chunked | ESP32原生支持 |
| ASR | 云服务器 Whisper.cpp 流式 | 中文识别准确 |
| 意图理解 | 云服务器 Hermes Agent | 复用现有能力 |
| TTS | Edge-TTS / CosyVoice | 免费, 中文自然 |
| 屏幕刷新 | MQTT JSON + 局部刷新 | 轻量, 不闪屏 |
| OTA分区 | A/B双区 (4MB Flash) | 安全回滚 |
| OTA下载缓冲 | PSRAM 512KB | 不擦写Flash |
| 固件托管 | 云服务器 HTTP | 复用Nginx |
| 升级触发 | 语音+自动检查+强制推送 | 全场景覆盖 |
| 显示/播报顺序 | 先显示后播报 | 视觉快反馈 |

## 音频扩展方案

默认使用 MAX98357A 驱动内置 4Ω 喇叭，但硬件预留两条扩展路径：

### 🎧 有线 AUX（PCM5102A）

```
ESP32 I2S ──┬── MAX98357A ── 内置喇叭 (默认)
             └── PCM5102A  ── 3.5mm ── 外接音箱 (后期补焊)
```

- PCM5102A 和 MAX98357A **共享同一条 I2S 总线**，ESP32 只发一路信号
- 用两个 GPIO 分别控制使能/静音，软件切换输出
- PCB 上预留 PCM5102A 焊盘 + 3.5mm 母座焊盘，不焊也不影响功能

### 📶 蓝牙音箱（ESP32 自带）

- ESP32 本身支持 BT Classic + A2DP Source
- **零硬件改动**，纯固件功能
- 固件实现：配对后切换音频输出到蓝牙，内置喇叭自动静音
- 注意：WiFi+BT 共用天线，蓝牙音乐期间 WiFi 吞吐会下降，语音流上传可能受影响
