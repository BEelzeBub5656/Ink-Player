## 电源树

```
Type-C 5V ──→ TP4056 ──→ DW01 ──→ 锂电池 2000mAh
                         │
                         ├──→ ME6211-3.3 ──→ ESP32 + 墨水屏 + INMP441 + MAX98357A
                         │
                         └──→ 5V直供 (Type-C插入时)
```

- **边充边用:** TP4056 支持负载共享，Type-C 插入时同时充电和供电
- **电池保护:** DW01 防过充/过放/短路
- **LDO:** ME6211 低压差 (200mV@300mA)，锂电池 3.7V→3.3V 无问题
- **CC下拉:** Type-C 母座 CC1/CC2 各接 5.1kΩ 到 GND，确保 C-to-C 线正常供电

## 音频扩展方案

### 🎧 有线 AUX（PCM5102A）

```
ESP32 I2S ──┬── MAX98357A ── 内置喇叭 (默认)
             └── PCM5102A  ── 3.5mm ── 外接音箱 (后期补焊)
```

- PCM5102A 和 MAX98357A **共享同一条 I2S 总线**
- 用两个 GPIO 分别控制使能/静音，软件切换输出
- PCB 上预留 PCM5102A 焊盘 + 3.5mm 母座焊盘

### 📶 蓝牙音箱（ESP32 自带）

- ESP32 BT Classic + A2DP Source，零硬件改动
- WiFi+BT 共用天线，蓝牙音乐期间 WiFi 吞吐下降

## OTA 分区表 (4MB Flash)

| 分区 | 偏移 | 大小 | 用途 |
|------|------|------|------|
| Bootloader | 0x1000 | ~32KB | 二级引导 |
| Partition Table | 0x8000 | ~4KB | 分区定义 |
| NVS | 0x9000 | 24KB | WiFi凭证+配置 |
| OTAData | 0xF000 | 8KB | OTA状态标记 |
| Factory | 0x10000 | 1.4MB | 出厂固件(恢复用) |
| OTA_0 | 0x270000 | 1.2MB | 运行区 |
| OTA_1 | 0x3A0000 | 1.2MB | 更新区 |

> 单区 1.2MB 估算: IDF+驱动 800KB + ESP-SR模型 300KB + 余量 100KB。OTA 固件先下载到 PSRAM 512KB 缓冲再写入。

## 功耗策略: ULP VAD

```
INMP441 ──→ I2S ──→ ULP 协处理器 (持续, ~2mA)
                        │
                    声能 > 阈值? ──No──→ 继续监听
                        │
                       Yes
                        │
                    唤醒 ESP32 主CPU
                        │
                    ESP-SR 唤醒词确认
                        │
                    匹配"小智"? ──No──→ 主CPU休眠
                        │
                       Yes
                        │
                    录音 → 上传 → AI 交互
                        │
                    完成后主CPU休眠
```

| 模式 | 功耗 | 说明 |
|------|------|------|
| ULP 监听 | ~2mA | ULP 只做声能检测, 不跑唤醒词 |
| 主CPU待机 | ~0.5mA | 屏幕保持, WiFi休眠 |
| 语音交互 | ~150mA | 录音+上传+播放, 每次~5秒 |
| **预估续航** | **~40天** | 2000mAh, 每天10次交互 |

## 功耗策略: ULP VAD

INMP441 → I2S → ULP 协处理器 (持续, ~2mA)
  → 声能 > 阈值? → 是 → 唤醒 ESP32 主CPU
  → ESP-SR 确认唤醒词"小智"
